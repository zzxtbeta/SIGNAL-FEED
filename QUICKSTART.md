# GRAVAITY 认知引擎 - 快速开始

## 🚀 启动应用

```bash
cd quantum-engine
npm install
npm run dev
```

应用将在 `http://localhost:3001` 启动

## ✨ 已完成的核心功能

### 1. 全局搜索 (Navbar)
- **功能**: 在顶部导航栏输入关键词，按回车搜索
- **搜索范围**: 信号标题、摘要、类型
- **效果**: 自动跳转到信号流页面并显示搜索结果
- **提示**: 搜索结果会显示匹配数量

### 2. 信号详情弹窗 - 完整交互
点击任意信号卡片打开详情弹窗，包含：

#### a) "为什么重要" 说明 (US02)
- 根据优先级自动生成重要性分析
- 高优先级：融资金额、团队背景、政策相关性
- 中优先级：技术突破、期刊影响力
- 低优先级：常规变更提示

#### b) 相关信号标签筛选
- **5个分类标签**: 论文、融资、工商、人员、政策
- **实时筛选**: 点击标签即时过滤相关信号
- **动态计数**: 每个标签显示该类型信号数量
- **可点击**: 点击相关信号可查看详情（当前为演示提示）

#### c) 三大操作按钮 - 真实功能
1. **记录为笔记**
   - 自动创建笔记，包含信号标题和摘要
   - 笔记保存到"我的笔记"页面
   - Toast提示操作成功

2. **加入关注**
   - 自动提取公司名称
   - 添加到"我的关注"页面
   - 防重复添加（已关注会提示）
   - Toast提示操作成功

3. **Chat 分析**
   - 关闭弹窗并准备打开Chat
   - 当前为演示提示（Chat功能待后端接入）

### 3. 我的关注 (MyFocus)
- **数据持久化**: 使用React Context全局状态管理
- **分类筛选**: 全部、公司、人物、技术
- **动态更新**: 从信号详情添加的关注会实时显示
- **初始数据**: 包含3个预设关注对象

### 4. 我的笔记 (MyNotes)
- **数据持久化**: 使用React Context全局状态管理
- **搜索功能**: 可搜索标题、内容、标签
- **动态更新**: 从信号详情创建的笔记会实时显示
- **初始数据**: 包含3篇预设笔记

### 5. 五类信号源 - 完整覆盖
Mock数据包含20条高质量信号，分布如下：
- **论文/专利**: 6条 (Nature, Science, PRL等顶刊)
- **融资信息**: 4条 (本源量子C轮、图灵量子A轮等)
- **工商动态**: 3条 (注册资本变更、专利申请等)
- **人员变迁**: 4条 (高管加入、团队创业等)
- **行业政策**: 3条 (科技部专项、地方政策等)

所有信号基于真实中国量子科技行业信息编写。

## 🔄 数据流说明

### 全局搜索流程
1. 用户在Navbar输入关键词 → 按回车
2. 通过URL参数传递搜索词 (`?search=xxx`)
3. SignalFeed读取URL参数
4. 过滤signals数组并显示结果

### 添加关注流程
1. 用户点击"加入关注"按钮
2. SignalDetailModal提取公司名称
3. 调用`addFocusItem()`添加到Context
4. MyFocus页面自动更新显示

### 创建笔记流程
1. 用户点击"记录为笔记"按钮
2. SignalDetailModal生成笔记对象
3. 调用`addNote()`添加到Context
4. MyNotes页面自动更新显示

## 📝 开发说明

### Mock/Real API切换
通过环境变量控制：
```bash
# .env.development
VITE_USE_MOCK_DATA=true

# .env.production
VITE_USE_MOCK_DATA=false
```

### 添加新信号
编辑 `src/mock/signals.ts`，按照现有格式添加：
```typescript
{
  id: 'signal-xxx',
  title: '信号标题',
  type: '论文', // 论文|融资|工商|人员|政策
  source: '来源',
  timestamp: '2026-02-06',
  priority: 'high', // high|mid|low
  summary: '详细摘要...',
  relatedEntities: { companies: 1, people: 2, technologies: 1 },
}
```
